# FT4222 USB-to-SPI 与 DAC8568 ±10V 模块硬件说明

## 目标与范围

本文档只覆盖当前仓库里的最小硬件链路：

- `Host PC`
- `FT4222 USB-to-SPI 模块`
- `DAC8568 ±10V SPI-to-DAC 模块`
- 后级模拟输入设备（例如振镜驱动板）

不覆盖：

- 振镜控制闭环
- 坐标标定
- `/gimbal/*` 软件接口
- GPIO 扩展控制（`LDAC`、`CLR`、复位脚等）

当前目标只是把 `FT4222 -> DAC8568` 这段链路稳定打通，能可靠输出指定模拟电压。

## 当前链路定位

当前推荐链路是：

```text
Host PC
-> FT4222 USB-to-SPI bridge
-> DAC8568 ±10V module
-> 模拟电压输出
-> 后级设备（如振镜驱动输入）
```

如果后级设备是模拟振镜驱动板，则当前推荐链路进一步具体化为：

```text
Host PC
-> FT4222 USB-to-SPI
-> DAC8568 ±10V board
-> galvo driver analog inputs (X/Y)
-> galvo mirrors
```

仓库中的 `FT4222` 封装位于：

- `src/io/ft4222_spi.hpp`
- `src/io/ft4222_spi.cpp`

它负责把 PC 侧的用户态程序转换为 SPI 主机写入；它不是 Linux `spidev`，也不是内核态 SPI 控制器。

## 器件角色

### FT4222 USB-to-SPI 模块

#### 模块简介

`FT4222H` 是 FTDI 的 USB 桥接芯片，常见成品模块会把它做成一个 `USB2SPI/USB2I2C` 小板：

- 上行通过 USB 接 PC
- 下行提供 `SPI`、`I2C`、部分 `GPIO` 能力

在本项目里，它只承担一个角色：把主机程序发出的 SPI 写命令转换成真实的 SPI 引脚时序。

#### 当前项目中实际用到的能力

- 通过 USB 接入主机
- 对外提供 SPI 主机信号：`CS/SYNC`、`SCLK`、`MOSI`
- 当前最小链路下不需要 `MISO`
- Linux 下走 `D2XX + LibFT4222` 用户态库，不走 `/dev/spidev*`

当前仓库还没有用到它的这些能力：

- `I2C`
- `GPIO` 扩展
- SPI 从机模式
- 多片选复杂切换

#### 你在硬件上真正需要关心什么

如果你用的是现成 `FT4222 USB2SPI` 模块，而不是裸 `FT4222H` 芯片设计，那么第一次联调通常只需要关心：

- 模块上哪个脚是 `CS`
- 哪个脚是 `SCLK`
- 哪个脚是 `MOSI`
- 哪个脚是 `GND`
- 模块逻辑电平是否为 `3.3V`
- 主机侧能否识别到 `0403:601c FT4222`

像晶振、`RREF`、`VCCIO`、裸芯片 `SS` 上拉等底层设计，通常已经在模块板上处理好了。

#### 当前软件封装边界

当前仓库中的 `Ft4222Spi` 封装只覆盖最小 SPI 打通能力：

- 枚举并打开 FT4222
- 初始化单线 SPI 主机
- 写一段字节流
- 做一次全双工传输

它还没有提供 DAC8568 专用 helper，也没有接管 `LDAC` / `CLR` 一类 GPIO 控制。

### DAC8568 ±10V 模块

#### 模块简介

你手里的这块板不是“裸 `DAC8568` 芯片最小系统”，而是一个已经做好的 `SPI -> 多通道模拟电压` 成品模块。它通常由几部分组成：

- `DAC8568` 八通道 16-bit DAC 芯片
- 参考电压与寄存器控制链路
- 把 DAC 原始输出变成 `±10V` 的后级模拟放大/偏置电路
- 电源端子、SPI 控制排针、模拟输出端子

因此你在使用时面对的是“成品输出模块”，不是单独一颗只讨论芯片寄存器的 DAC。

#### 当前模块在链路中的职责

- 左上角两位端子：模块电源输入 `+5V / GND`
- 左侧黄排针：数字控制接口
- 右侧端子排：模拟输出 `A~H` 与地
- 板上默认说明为：内部 `2.5V` 参考，经过后级模拟电路转换为 `±10V` 输出范围

它在整条链路里的职责是：

- 接收来自 FT4222 的 SPI 控制帧
- 将数字码值写入对应通道寄存器
- 通过板上模拟链路输出 `A~H` 八路模拟电压

#### 裸芯片与成品模块的区别

这里有一个容易混淆的点：

- 裸 `DAC8568` 芯片官方文档讨论的是芯片级 SPI 协议、参考电压、寄存器和原始 DAC 行为
- 你这块 `±10V` 模块在裸芯片基础上又增加了供电与模拟调理电路，所以最终表现为“写 SPI 码值，得到 ±10V 输出”

也就是说，联调时要同时遵守两类约束：

- `DAC8568` 芯片的 SPI 时序约束
- 成品模块的 `5V` 供电、端子定义和 `±10V` 输出约束

#### 你在硬件上真正需要关心什么

第一次联调这块 DAC 模块时，最重要的是：

- 模块必须单独接 `5V`
- 数字控制口按 `3.3V` 逻辑看待
- 只需要 `SYNC/SCLK/DIN/GND` 就能完成最小 bring-up
- 右侧输出 `A~H` 的端子顺序不是简单从上到下 `A/B/C/...`
- 这块板输出的是 `±10V`，而不是裸 DAC 那种直接 `0~2.5V` 或 `0~5V`

#### 当前 bring-up 建议

从 bring-up 角度，建议先把它当作一个“只写、不读、先测单通道”的 SPI 外设：

- 先只测 `A` 通道
- 先打 `0V`
- 再打 `+1V`
- 再打 `-1V`
- 先不用 `LDAC`、`CLR`、外部 `VREF`

## DAC8568 模块接口说明

### 电源接口

- `+5V`
  - 模块供电正端
- `GND`
  - 模块供电地

注意：

- 该模块需要独立 `5V` 供电
- 不要超过 `5.5V`
- 反接或过压都可能损坏模块

### 数字控制接口

从图片与模块手册可见，排针包含：

- `VCC`
- `GND`
- `SCLK`
- `DIN`
- `CLR`
- `SYNC`
- `LDAC`

其中当前最小链路真正需要的只有：

- `GND`
- `SCLK`
- `DIN`
- `SYNC`

其它脚位含义：

- `CLR`
  - 下降沿清除代码寄存器
  - 当前不接
- `LDAC`
  - 控制寄存器值装载到 DAC 输出寄存器
  - 当前不接
- `VREF`
  - 外部参考输入
  - 当前不接，先使用模块默认内部参考
- 数字排针 `VCC`
  - 模块手册未清晰说明其输入/输出角色
  - 模块示例接线中未使用此脚
  - 当前建议先不接，避免把逻辑参考脚误当成供电脚

### 输出端子顺序

模块右侧端子从上到下为：

- `GND`
- `B`
- `D`
- `F`
- `H`
- `G`
- `E`
- `C`
- `A`
- `GND`

也就是说：

- `A` 通道在较下方
- `B` 通道在较上方
- 两端都有地，方便接表或接后级设备

## FT4222 到 DAC8568 的推荐接线

第一次打通时，按下面最小接线即可：

| FT4222 模块 | DAC8568 模块 | 是否必须 | 说明 |
|---|---|---:|---|
| `GND` | 电源口 `GND` | 是 | 共地 |
| `GND` | 数字口 `GND` | 是 | 与 SPI 逻辑共地 |
| `SS0O` / `CS` | `SYNC` | 是 | 片选，低有效 |
| `SCLK` | `SCLK` | 是 | SPI 时钟 |
| `MOSI` / `DO` | `DIN` | 是 | SPI 主发从收 |
| `MISO` | 不接 | 否 | 当前模块无读回链路 |
| 任意 GPIO | `CLR` | 否 | 初版不接 |
| 任意 GPIO | `LDAC` | 否 | 初版不接 |
| 外部参考 | `VREF` | 否 | 初版不接 |
| 3.3V/5V | 数字口 `VCC` | 否 | 初版不接 |

### 供电连接

DAC 模块还必须额外接电源：

| 外部电源 | DAC8568 模块 |
|---|---|
| `+5V` | 电源口 `+5V` |
| `GND` | 电源口 `GND` |

注意：

- `FT4222` 的 USB 侧供电不能替代 DAC 板专用 `+5V` 输入
- 必须保证 `FT4222 GND`、`DAC 电源 GND`、`DAC 数字 GND` 共地

## 不要这样接

第一次 bring-up 时，避免下面这些错误：

- 不要把 FT4222 的 `MISO` 接到 DAC 的任意控制脚
- 不要把 DAC 数字排针里的 `VCC` 直接当成 `+5V` 供电脚
- 不要只接 SPI 线不共地
- 不要把 `5V` 数字逻辑直接打到 `SCLK`/`DIN`/`SYNC`
- 不要把 DAC 板直接接到振镜电机线；它接的是后级模拟输入，不是功率驱动输出

## SPI 时序与协议约束

根据 DAC8568 官方资料与模块说明，当前应按下面理解：

- 接口类型：`3-wire SPI`
- 使用信号：`SYNC`、`SCLK`、`DIN`
- `SYNC`：低有效
- 传输方向：只写
- 帧长：`32 bit`，即每次完整命令必须发 `4 byte`
- 位序：`MSB first`
- 数据采样边沿：`SCLK` 下降沿
- 最高速率：资料标称可到 `50 MHz`，首次联调建议从低速开始

对于 FT4222，首次联调建议：

- 单线 SPI：`SPI_IO_SINGLE`
- `CPOL = 0`
- `CPHA = 1`
- 片选使用 `SS0O`
- 时钟先从约 `1 MHz` 左右开始，再逐步提速

在当前仓库封装里，推荐起步配置是：

```cpp
Ft4222Config{
    .clock_div = Ft4222SpiDiv::kDiv64,
    .cpol = Ft4222Cpol::kIdleLow,
    .cpha = Ft4222Cpha::kTrailing,
    .cs_channel = 0,
}
```

## 上电与联调顺序

推荐顺序：

1. 断电状态下完成所有接线
2. 检查 DAC 板 `+5V/GND` 极性
3. 检查 `SYNC/SCLK/DIN/GND` 是否对应正确
4. 确认 FT4222 已经通过 USB 接入主机
5. 给 DAC 板上 `5V`
6. 主机侧确认能枚举到 FT4222
7. 程序打开 FT4222，并发送 SPI 命令
8. 用万用表测量 `A~H` 输出对 `GND` 的电压

Linux 下可先用：

```bash
lsusb | rg 0403:601c
```

若看到 `0403:601c FT4222`，说明 USB 枚举已完成。

## 输出电压与码值

模块手册给出的 `±10V` 输出换算公式可按下面理解：

```text
D = (V + 10) / 20 * 65535
V = D / 65535 * 20 - 10
```

其中：

- `D`
  - 16-bit DAC code
- `V`
  - 目标输出电压，单位 V

对应近似关系：

- `0x0000` -> `-10V`
- `0x7FFF / 0x8000` -> 约 `0V`
- `0xFFFF` -> `+10V`

首次联调建议先打这几个点：

- `0V`
- `+1V`
- `-1V`

不要第一次就直接打满量程，便于检查接线和偏置。

## 当前仓库软件侧约束

仓库已经有 FT4222 的最小封装，但当前有几个重要限制：

### 1. DAC8568 不能用 `2 byte` 命令

`README.md` 中的 FT4222 示例写的是 `2 byte` 示例数据，这对于 `DAC8568` 不够。

原因：

- DAC8568 一次命令要求 `32 bit`
- 必须在同一次 SPI 事务里发完 `4 byte`
- 如果拆成多个小事务，`SYNC` 会在字节间抬高，DAC 会把命令判为无效或截断

### 2. 当前封装现在会真实下发 `sys_clock` 和 `cs_active`

当前 `src/io/ft4222_spi.cpp` 已经会调用：

- `FT4222_SPIMaster_Init(...)`
- `FT4222_SetClock(...)`
- `FT4222_SPIMaster_SetCS(...)`
- `FT4222_GetMaxTransferSize(...)`

因此：

- `Ft4222Config.sys_clock` 不再只是本地字段，而会真实下发到 FT4222
- `Ft4222Config.cs_active` 也会真实设置片选极性
- `negotiated_clock_hz()` 现在和配置含义一致，能作为估算值使用

### 3. 当前仍然缺少 DAC8568 专用 helper

虽然 FT4222 最小封装已经能稳定打开设备并发送 SPI 写命令，但当前仓库仍然没有 DAC8568 专用 helper 来统一封装：

- `32-bit` 命令组帧
- 通道选择 `A~H`
- `±10V` 电压到 16-bit 码值换算
- 内部参考初始化
- 安全测试序列

当前这些逻辑主要由 `tool_dac8568_smoke` 承担。

## 最小软件 bring-up 建议

第一次打通时，建议软件侧做两件事：

1. 显式打开 DAC 内部参考
2. 向单个通道写一个小电压值

仓库里现在已经提供对应硬件测试入口：

```bash
./build/tool_dac8568_smoke --help
./build/tool_dac8568_smoke --channel A --voltage 0 --hold-ms 0
./build/tool_galvo_smoke --help
```

常见的第一条参考使能序列可写成：

```text
0x08000001
```

随后再发送完整 `32-bit` 通道写命令。

如果后续要正式把 DAC8568 接进仓库，建议新增一个专用 helper，负责：

- 组装 `32-bit` 帧
- 通道选择 `A~H`
- 电压到码值换算
- 内部参考初始化

而不是直接在业务逻辑里裸写字节数组。

## 常见故障排查

### USB 已接入，但程序找不到 FT4222

先查：

- `lsusb` 是否能看到 `0403:601c`
- `libft4222.so` 是否存在且能被程序加载
- 是否有权限问题

### SPI 有波形，但 DAC 没输出

优先查：

- 是否给 DAC 板单独接了 `+5V`
- 是否共地
- 是否发了完整 `4 byte`
- SPI 模式是否为 `CPOL=0, CPHA=1`
- 是否需要先显式使能内部参考

## 振镜驱动输入接线

你给出的振镜参数：

- 扫描速度：`23-25Kpps ILDA @ 8°`
- 扫描角度：`±30°`
- 输入信号：`±5V`
- 电源：`±15V 1A`

这类参数与常见 `ILDA`/模拟振镜驱动板高度一致。外部资料里，这一类驱动最常见的是：

- 模拟输入为 `±5V`
- 信号输入为差分 `X+ / X- / Y+ / Y-` 或每轴 `+ / GND / -`
- 驱动板单独使用 `+15V / GND / -15V` 电源

因此，当前推荐先按“差分模拟输入”来接。

### 1. 差分接法（推荐）

如果你的振镜驱动输入端有这些标签之一：

- `X+`, `X-`, `Y+`, `Y-`, `GND`
- `X+`, `SGND`, `X-` 与 `Y+`, `SGND`, `Y-`
- `ILDA X+`, `ILDA X-`, `ILDA Y+`, `ILDA Y-`

则推荐使用 DAC8568 的四路输出直接组成差分对：

| DAC8568 输出 | 振镜驱动输入 | 说明 |
|---|---|---|
| `A` | `X+` | X 轴正向输入 |
| `C` | `X-` | X 轴反向输入 |
| `B` | `Y+` | Y 轴正向输入 |
| `D` | `Y-` | Y 轴反向输入 |
| `GND` | `Signal GND` / `SGND` | 信号参考地 |

推荐原因：

- 能直接合成差分输入
- 更接近 `ILDA` 风格输入
- 抗干扰更好
- 不需要额外运放就能生成互为反相的 X/Y 信号

### 2. 单端接法（兼容）

如果你的振镜驱动板只有：

- `X`, `Y`, `GND`

或者文档明确说明它接受单端 `±5V` 指令输入，则可先按单端方式接：

| DAC8568 输出 | 振镜驱动输入 | 说明 |
|---|---|---|
| `A` | `X` | X 轴输入 |
| `B` | `Y` | Y 轴输入 |
| `GND` | `GND` | 信号地 |

此时：

- `C`、`D` 不接
- 运行测试工具时使用 `--wiring single-ended`

### 3. 电源接法

振镜驱动板的供电不要接到 DAC 板上，必须单独供电：

| 电源 | 振镜驱动板 |
|---|---|
| `+15V` | `+15V` / `+VCC` |
| `GND` | `GND` |
| `-15V` | `-15V` / `-VEE` |

注意：

- 振镜驱动电源和 DAC 板是两套功能，不要混供
- 但 DAC 信号地与振镜驱动信号地必须共参考
- 不要带电插拔振镜驱动线、振镜电机线或信号线

### 4. 第一次上电前的接线检查

第一次接振镜信号前，先确认：

1. `FT4222 -> DAC8568` 已经通过 `tool_dac8568_smoke` 验证
2. 振镜驱动电源极性正确：`+15V / GND / -15V`
3. 信号线已经共地
4. 差分模式下 `X+`/`X-`、`Y+`/`Y-` 没接反
5. 默认先用很小的测试幅度，不要一上来就接近满量程

### 5. 仓库里的振镜信号测试工具

仓库现在提供：

```bash
./build/tool_galvo_smoke --help
```

它支持三类目标：

- 静态/阶跃测试：`center`、`xp`、`xn`、`yp`、`yn`
- 正弦扫频：`xsine`、`ysine`
- XY 图形轨迹：`xysine`

它的默认差分映射是：

- `X+ = A`
- `X- = C`
- `Y+ = B`
- `Y- = D`

默认测试序列是：

```text
center
-> X 正方向小幅偏转
-> center
-> X 负方向小幅偏转
-> center
-> Y 正方向小幅偏转
-> center
-> Y 负方向小幅偏转
-> center
```

默认测试电压是很小的每腿命令电压：

- `0.5V` on `+` and `-0.5V` on `-`

也就是说，差分模式下振镜驱动实际看到约 `1.0V` 的差分命令。这比满量程小很多，适合第一次 bring-up。

### 6. 第一次推荐测试命令

如果你的驱动输入是差分：

```bash
./build/tool_galvo_smoke --target center --hold-ms 0
./build/tool_galvo_smoke --diff-voltage 0.5 --hold-ms 1500
```

X 轴正弦扫频示例：

```bash
./build/tool_galvo_smoke \
  --target xsine \
  --diff-voltage 2.5 \
  --sweep-start-hz 0.5 \
  --sweep-end-hz 3.0 \
  --sweep-duration-ms 3000 \
  --sample-hz 100
```

上面这个例子在差分模式下表示：

- `X+` 每腿峰值 `+/-2.5V`
- `X-` 每腿峰值 `-/+2.5V`
- 驱动实际看到 `±5V` 的等效差分正弦输入

XY 正弦图形示例：

```bash
./build/tool_galvo_smoke \
  --target xysine \
  --diff-voltage 2.5 \
  --curve-cycles 1 \
  --sweep-start-hz 0.5 \
  --sweep-end-hz 1.5 \
  --sweep-duration-ms 4000 \
  --sample-hz 120
```

这个模式会：

- 用 `X` 轴做往返水平扫描
- 用 `Y` 轴输出 `sin(x)`
- 形成一个连续的正弦图形轨迹
- 在差分模式下输出等效 `±5V` 峰值图形命令

如果你的驱动输入是单端：

```bash
./build/tool_galvo_smoke --wiring single-ended --target center --hold-ms 0
./build/tool_galvo_smoke --wiring single-ended --diff-voltage 0.5 --hold-ms 1500
```

### 7. 为什么默认不用大电压

振镜是闭环高速机械系统，不适合第一次就上大幅度阶跃。默认小电压的原因是：

- 避免第一次接线错误时直接大幅甩镜
- 避免接反或极性错误时把镜子打到机械极限附近
- 先验证方向、中心、响应是否正常

如果后续需要更大角度，再逐步增加 `--diff-voltage`。对于差分模式，`--diff-voltage 1.0` 表示 `+1V / -1V` 每腿输出，驱动实际看到 `2V` 差分命令。

### 8. 常见接线判断

如果你看到驱动板是下面这种接口风格，可以这样判断：

- `X+ / X- / Y+ / Y- / GND`
  - 按差分接法
- `X+ / SGND / X-` 和 `Y+ / SGND / Y-`
  - 仍按差分接法
- `X / Y / GND`
  - 按单端接法

如果丝印完全不明确，先不要盲接满套信号，先拍驱动板输入端子图再确认。

### 输出电压不对或打不满

优先查：

- 码值换算是否按 `±10V` 模块公式处理
- 是否存在零点偏置，需要逐通道校准
- 后级负载是否过重

### 电压变化方向相反

优先查：

- 测量点是否接错通道
- 上位机的码值换算是否把 `-10V ~ +10V` 写反

## 当前推荐的第一次联调方案

第一次接通时，推荐只做下面这条最小链路：

1. DAC 板接独立 `5V`
2. FT4222 只接 `GND/SYNC/SCLK/DIN`
3. 不接 `MISO/CLR/LDAC/VREF/VCC`
4. 用万用表只测 `A` 通道对 `GND`
5. 先发 `0V`
6. 再发 `+1V`
7. 再发 `-1V`

这个方案最容易定位问题，也最不容易因为外围复杂度把问题放大。
